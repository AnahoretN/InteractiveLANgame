## Введение

### Назначение приложения

**InteractiveLANgame** — это многопользовательская викторина в стиле "Своя игра" (Jeopardy), предназначенная для проведения локальных и онлайн-игровых сессий.

### Ключевые особенности

- **P2P-соединение** — прямое соединение между хостом и игроками через WebRTC
- **LAN/Internet режимы** — работа в локальной сети или через интернет
- **Мобильные клиенты** — игроки подключаются со своих смартфонов в качестве пультов
- **Редактор пакетов** — создание собственных наборов вопросов с медиа-контентом
- **Система таймеров** — фазы чтения и ответа с настраиваемым временем
- **Система ставок** — специальный раунд Super Game с механикой ставок
- **QR-код подключение** — быстрое подключение сканированием QR-кода

### Потоки управления

#### Маршрутизация приложения

```
App.tsx
    ├── URL hash = #/mobile → MobileView
    └── иначе → HostView
```

#### Жизненный цикл хоста

```
1. Запуск приложения
2. Генерация Host ID
3. Настройка IP (LAN) или Internet режим
4. Создание команд
5. Генерация QR-кода
6. Подключение игроков
7. Выбор пакета вопросов
8. Запуск игровой сессии
9. Проведение игры
10. Завершение (тройное ESC)
```

#### Жизненный цикл мобильного клиента

```
1. Сканирование QR-кода или ввод URL
2. Ввод имени
3. Выбор команды
4. Ожидание подтверждения хоста
5. Игровой экран с буззером
6. Нажатие буззера при активном вопросе
7. Отображение результата
```

## Интерфейс Хоста

### Главный экран (HostView)

#### Панель настройки хоста (HostSetupPanel)

**Элементы:**

1. **Поле ввода IP**
   - Назначение: Указывает IP-адрес для LAN режима
   - Автозаполнение: Определяет локальный IP автоматически
   - Формат: `192.168.x.x`

2. **Переключатель LAN/Internet**
   - LAN: Требует signalling server на локальном IP
   - Internet: Использует публичный signalling server

3. **Session ID**
   - 5-символьный идентификатор сессии
   - Отображается крупным шрифтом
   - Кнопка обновления для генерации нового ID

4. **QR-код**
   - Содержит данные для подключения
   - Включает IP и session ID
   - Анимированный QR-код

5. **Кнопка "Скопировать ссылку"**
   - Копирует URL подключения в буфер обмена
   - Визуальная анимация при копировании

#### Панель лобби (LobbyPanel)

**Функционал:**

1. **Список команд**
   - Отображение всех созданных команд
   - Счётчик игроков в каждой команде
   - Визуальная индикация буззеров (белая вспышка)
   - Удаление команд

2. **Создание команды**
   - Инлайн-форма для быстрого создания
   - Валидация имени (не пустое)
   - Автоматическое удаление пустых команд через 5 минут

3. **Список подключённых клиентов**
   - Имя игрока
   - Текущая команда
   - Статус подключения
   - Drag-and-drop для распределения по командам

4. **Кнопка настроек**
   - Открывает модальное окно Session Settings
   - Значок шестерёнки

5. **Кнопка выбора пакета**
   - Открывает GameSelectorModal
   - Отображает выбранный пакет

6. **Кнопка "Начать сессию"**
   - Активна при наличии подключённых клиентов
   - Запускает игровой процесс

- `1-9` — быстрый переход к раунду

- `↑/↓` — прокрутка тем

#### 3. Question Modal (Модальное окно вопроса)

**Элементы:**

1. **Заголовок**
   - Название темы
   - Стоимость вопроса
   - Название раунда

2. **Текст вопроса**
   - Адаптивный размер шрифта
   - Автоматическая подстройка под длину
   - Поддержка форматирования

3. **Медиа-контент**
   - Изображения: центрированные, макс. ширина 80%
   - Видео: автоплей, контролы
   - Аудио: кнопка воспроизведения

4. **Таймер**
   - Фаза чтения: обратный отсчёт
   - Фаза ответа: обратный отсчёт
   - Цветовая индикация:
     - 🟢 Зелёный — достаточно времени
     - 🟡 Жёлтый — менее 50%
     - 🔴 Красный — менее 20%

5. **Кнопки управления ответом**
   - `Правильно` (Space) — добавить очки
   - `Неправильно` (Ctrl) — вычесть очки
   - `Пропустить` (Esc) — без изменения очков

6. **Панель ответов**
   - Список вариантов ответов
   - Клик для подсветки правильного

**Фазы Super Game:**

| Фаза | Описание | Действия |
|------|----------|----------|
| Select Super Themes | Выбор тем для Super Game | Выбрать 1-3 темы |
| Place Bets | Команды делают ставки | Мониторинг ставок |
| Super Question | Показ вопроса | Включить аудио/видео |
| Super Answers | Команды отвечают | Мониторинг ответов |

**Механика ставок:**
- Минимальная ставка: 1
- Максимальная ставка: текущий счёт команды

**Начисление очков:**
- Правильный ответ: +ставка
- Неправильный ответ: -ставка

### Система таймеров

#### Reading Timer (Таймер чтения)

**Назначение:** Время для ведущего прочитать вопрос

**Расчёт:**
```
Reading Time = (Длина текста / Количество букв) × Reading Time Per Letter
```

**Настройка:**
- `readingTimePerLetter`: 0.05–2.0 секунд на букву
- Может быть отключён (бесконечное время)

#### Response Timer (Таймер ответа)

**Назначение:** Окно для нажатия буззера игроками

**Настройка:**
- `responseWindow`: 1–60 секунд
- Обратный отсчёт с визуальной индикацией

#### Handicap System (Система гандикапа)

**Принцип:** Команды с меньшим счётом получают преимущество

**Формула:**
```
Handicap Delay = Base Delay × (1 - TeamScore / MaxScore)
```

**Пример:**
- Базовая задержка: 2 секунды
- Команда с 1000 очков: 0 сек задержки
- Команда с 500 очками: 1 сек задержки
- Команда с 0 очков: 2 сек задержки

### Система буззера

#### Обработка нажатий

**Поток событий:**

```
1. Игрок нажимает буззер
   ↓
2. Отправка события BUZZ с timestamp
   ↓
3. Хост получает событие
   ↓
4. Проверка: isFirstBuzz()?
   ├─ Да → Запись первого буззера
   └─ Нет → Проверка на одновременность
   ↓
5. Расчёт разницы во времени
   ↓
6. Определение победителя
   ↓
7. Отправка результата всем клиентам
```

#### Одновременные нажатия

**Обнаружение:**
- Порог: 0.25–2.0 секунды (настраиваемый)
- Если разница < порога → считаются одновременными

**Разрешение коллизий:**

| Ситуация | Результат |
|----------|-----------|
| Команды с равным счётом | Случайный выбор |
| С включённым Underdog Bonus | Преимущество отстающей (+20%) |
| Collision Handling отключён | Первый по времени |

**Формула Underdog Bonus:**
```
Advantage = 1 - (TeamScore / LeaderScore) × 0.2
```

### Система очков

#### Начисление очков

| Действие | Очки |
|----------|------|
| Правильный ответ | +Стоимость вопроса |
| Неправильный ответ | -Стоимость вопроса (опционально) |
| Пропуск | 0 |

#### Хранение очков

```typescript
interface TeamScore {
  teamId: string;
  teamName: string;
  score: number;
}
```

**Синхронизация:**
- Очки обновляются на хосте
- Рассылаются всем клиентам через STATE_SYNC
- Отображаются в реальном времени

## Сетевое взаимодействие (P2P)

### Архитектура

```
┌─────────────┐         signalling          ┌─────────────┐
│             │ ◄────────────────────────► │             │
│   Хост      │      (WebSocket/9000)      │  Клиент 1   │
│  (Host)     │ ◄─────────┬───────────────► │  (Mobile)   │
└─────────────┘           │                 └─────────────┘
                            │
                            │ WebRTC DataChannel
                            │ (P2P)
                            ▼
                     ┌─────────────┐
                     │  Клиент 2   │
                     │  (Mobile)   │
                     └─────────────┘
```

### Режимы подключения

#### LAN Mode

**Требования:**
- Хост и клиенты в одной сети
- Запущен signalling server
- Хост знает свой IP

**Подключение:**
```
ws://192.168.1.100:9000
```

**QR-код содержит:**
```
{"mode":"lan","ip":"192.168.1.100","port":9000,"sessionId":"ABC12"}
```

#### Internet Mode

**Требования:**
- Доступ к публичному signalling server
- Стабильное интернет-соединение

**Подключение:**
```
wss://public-signalling-server.com
```

**QR-код содержит:**
```
{"mode":"internet","sessionId":"ABC12"}
```

### Типы сообщений

#### STATE Messages (Гарантированная доставка)

| Тип | Платежная нагрузка | Описание |
|-----|-------------------|----------|
| JOIN_TEAM | clientId, teamId | Игрок присоединяется к команде |
| LEAVE_TEAM | clientId | Игрок покидает команду |
| TEAM_CONFIRMED | clientId | Подтверждение хоста |
| UPDATE_SCORE | teamId, score, delta | Обновление счёта |
| BUZZER_STATE | active, timerPhase | Состояние буззера |

#### EVENT Messages (Низкая задержка)

| Тип | Платежная нагрузка | Описание |
|-----|-------------------|----------|
| BUZZ | clientId, buzzTime | Нажатие буззера |
| SUPER_GAME_BET | teamId, bet | Ставка в Super Game |
| SUPER_GAME_ANSWER | teamId, answer | Ответ в Super Game |

#### SYNC Messages (Периодическая синхронизация)

| Тип | Платежная нагрузка | Описание |
|-----|-------------------|----------|
| STATE_SYNC | buzzerState, teams | Полная синхронизация |
| TEAMS_SYNC | teams | Список команд |
| COMMANDS_LIST | commands | Список команд |

#### CONTROL Messages (Управление соединением)

| Тип | Платежная нагрузка | Описание |
|-----|-------------------|----------|
| HANDSHAKE | clientId, protocolVersion | Первичное рукопожатие |
| HANDSHAKE_RESPONSE | hostId, sessionVersion | Ответ хоста |
| PING | timestamp | Проверка соединения |
| PONG | originalTimestamp | Ответ на пинг |

### Качество соединения

**Метрики:**

```typescript
interface ConnectionQuality {
  rtt: number;           // Round-trip time (ms)
  packetLoss: number;    // % потерянных пакетов
```

## Устранение неполадок

### Подключение

**Проблема:** Клиенты не могут подключиться

**Решения:**
1. Проверить, что signalling server запущен (порт 9000)
2. Проверить firewall — порт 9000 должен быть открыт
3. Убедиться, что хост и клиенты в одной сети (LAN режим)
4. Попробовать Internet режим

**Проверка signalling server:**
```bash
# Проверить, запущен ли процесс
netstat -an | grep 9000

# Или использовать PowerShell
Get-NetTCPConnection -LocalPort 9000
```

### Буззер не работает

**Проблема:** Нажатие буззера не регистрируется

**Решения:**
1. Проверить статус подключения (зелёный индикатор)
2. Убедиться, что буззер активен (пульсирует)
3. Проверить задержку соединения (RTT)
4. Перезапустить мобильный клиент

### Таймеры не синхронизированы

**Проблема:** Таймер на хосте и клиентах отличается

**Решения:**
1. Проверить синхронизацию времени на устройствах
2. Уменьшить readingTimePerLetter
3. Увеличить responseWindow

### Падение соединения

**Проблема:** Клиенты периодически отключаются

**Решения:**
1. Проверить качество WiFi сигнала
2. Уменьшить количество одновременных клиентов
3. Использовать проводное соединение для хоста
4. Проверить нагрузку на signalling server

### QR-код не сканируется

**Проблема:** Камера не распознаёт QR-код

**Решения:**
1. Увеличьте размер QR-кода (настройки)
2. Улучшите освещение
3. Попробуйте альтернативное приложение для сканирования
4. Используйте текстовую ссылку вместо QR
