# Media Streaming System

## Overview
Новая система трансляции медиафайлов с хоста на демо-экран через P2P соединение.

## Как это работает

### 1. Типы медиафайлов

Система поддерживает три типа медиа:

- **Локальные файлы из пакета**: Изображения, видео, аудио файлы, упакованные в игровой пакет
- **YouTube ссылки**: Ссылки на YouTube видео, которые конвертируются в embed формат
- **Внешние URL**: Прямые ссылки на медиафайлы в интернете

### 2. Процесс трансляции

#### Для локальных файлов:
1. Хост определяет, что медиа файл является локальным (blob URL)
2. Файл извлекается из IndexedDB
3. Конвертируется в base64
4. Передается через P2P как сообщение `MEDIA_TRANSFER`
5. Демо-экран получает base64 и создает blob URL

#### Для YouTube ссылок:
1. Хост определяет, что это YouTube ссылка
2. Конвертирует в embed формат
3. Передает только URL через `MEDIA_TRANSFER`
4. Демо-экран использует URL напрямую

#### Для внешних URL:
1. Хост определяет, что это внешняя ссылка
2. Передает только URL через `MEDIA_TRANSFER`
3. Демо-экран использует URL напрямую

### 3. Компоненты системы

#### Утилиты (`utils/mediaStream.ts`)
- `isYouTubeUrl()` - проверяет, является ли URL YouTube ссылкой
- `isBlobUrl()` - проверяет, является ли URL локальным файлом
- `detectMediaType()` - определяет тип медиа (image/video/audio/youtube)
- `createMediaInfo()` - создает информацию о медиа для передачи
- `fileToBase64()` - конвертирует файл в base64
- `base64ToBlobUrl()` - конвертирует base64 обратно в blob URL

#### Компонент MediaStreamer (`components/host/game/MediaStreamer.ts`)
- Автоматически транслирует медиафайлы при открытии вопроса
- Кэширует переданные файлы для предотвращения дублирования
- Работает без UI (невидимый компонент)

#### Типы сообщений (`types.ts`)
- `MediaTransferMessage` - сообщение для передачи медиа
- `MediaRequestMessage` - сообщение для запроса медиа

### 4. Интеграция в компоненты

#### GamePlay
- Компонент `MediaStreamer` добавлен в основной render
- Автоматически транслирует медиа при изменении `activeQuestion`

#### ScreenView
- Обрабатывает сообщения `MEDIA_TRANSFER`
- Кэширует полученные медиа в `window.mediaTransferCache`
- Использует кэшированные медиа при отображении вопросов

## Преимущества

1. **Автоматическая трансляция**: Медиа файлы передаются автоматически при открытии вопроса
2. **Поддержка всех типов**: Работает с локальными файлами, YouTube и внешними ссылками
3. **Оптимизация**: Кэширование предотвращает дублирование передач
4. **Надежность**: Fallback на IndexedDB для локальных файлов

## Использование

Система работает автоматически без дополнительной настройки:

1. Хост открывает вопрос с медиа
2. MediaStreamer автоматически передает медиа на демо-экран
3. Демо-экран получает и кэширует медиа
4. При отображении вопроса используется кэшированное медиа

## Отладка

Для отладки используйте консоль браузера:

- `[MediaStreamer]` - логи трансляции на хосте
- `[ScreenView]` - логи приема на демо-экране
- `window.mediaTransferCache` - кэш медиа на демо-экране

## Технические детали

### Ограничения P2P
- Максимальный размер сообщения: зависит от P2P провайдера
- Рекомендуется не передавать файлы больше 50МБ
- Для больших файлов рассмотрите альтернативные методы

### Производительность
- Base64 кодирование увеличивает размер на ~33%
- Локальные файлы конвертируются асинхронно
- YouTube и внешние URL не требуют конвертации

### Совместимость
- Работает с существующей системой медиа менеджмента
- Совместимо с IndexedDB хранилищем
- Не требует изменений в форматах пакетов
